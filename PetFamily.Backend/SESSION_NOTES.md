## Ключевые решения (важно)

1. Режим запуска
- API запускается локально через Visual Studio.
- Docker используется только для PostgreSQL через `docker-compose.yml`.

2. Решения по `Pet`
- `Status` пока оставлен строкой (`string`), к VO/enum вернуться позже.
- `Breed` переведён в `Guid` (шаг “на вырост” перед B-4.4).
- `Requisites` хранится как коллекция.
- `CreatedAt` задаётся внутри модели (`UtcNow`), не передаётся снаружи.
- При создании `Pet` сейчас обязательны `requisites` и `photos` (осознанное решение).

3. Формат работы
- Основной код пишет разработчик (студент).
- Codex — наставник и ревьюер: маленькие шаги, объяснения, без больших автоправок без явного запроса.
